# API

## Описание сущности Device (устройство)

1. Device
   1. ID – уникальный идентификатор устройства
   2. Name – человекочитаемое имя устройства
   3. Type – тип устройства (датчик, исполнительное устройство, камера, хаб)
   4. RoomID – идентификатор комнаты, к которой привязано устройство
   5. Status – текущий статус (online, offline, error)
   6. Configuration – JSON-объект с настройками устройства
   7. LastSeen – timestamp последней активности
   8. Manufacturer – производитель устройства
   9. Model – модель устройства

## Описание сущности Scenario (сценарий)

2. Scenario
   1. ID – уникальный идентификатор сценария
   2. Title – название сценария (например, "Вечерний релакс")
   3. Description – подробное описание сценария
   4. Trigger – условие запуска (время, событие, состояние устройства)
   5. Actions – список действий в формате JSON (какие устройства и как управлять)
   6. Enabled – флаг активности сценария (true/false)
   7. CreatedBy – идентификатор пользователя, создавшего сценарий
   8. CreatedDate – дата создания сценария

## Функции (эндпоинты)

1. CRUDS (create, read, update, delete, search) для устройств (Device)
2. CRUDS (create, read, update, delete, search) для сценариев (Scenario)
3. Дополнительные endpoints:
   - GET /api/devices/{id}/state – получить текущее состояние устройства
   - POST /api/scenarios/{id}/execute – выполнить сценарий немедленно
   - GET /api/rooms – список комнат с устройствами
   - POST /api/voice – обработка голосовой команды

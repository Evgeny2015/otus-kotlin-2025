# Тестовые сценарии для умного дома с ИИ

#### **FR-001: Адаптивное управление климатом**
| **Test ID** | T-001                                                                 |
|-------------|-----------------------------------------------------------------------|
| **Requirement** | FR-001                                                               |
| **Scenario** | Система автоматически регулирует температуру в гостиной на основе расписания пользователя |
| **Preconditions** | 1. Пользователь настроил расписание "Рабочий день" с 9:00 до 18:00<br>2. Датчик температуры показывает 22°C<br>3. Центральный хаб подключен к термостату |
| **Test Steps** | **Given:** Время 8:55, температура 22°C<br>**When:** Наступает 9:00<br>**Then:** Система устанавливает целевую температуру 20°C (рабочий режим)<br><br>**When:** Пользователь возвращается в 18:05<br>**Then:** Система повышает температуру до 23°C (комфортный режим) |
| **Test Data** | Расписание: Пн-Пт 9:00-18:00 рабочий режим (20°C), остальное время комфортный (23°C) |
| **Pass Criteria** | 1. Температура изменена в пределах ±0.5°C от целевой<br>2. Изменение произошло в течение 5 минут после времени расписания<br>3. Уведомление отправлено в мобильное приложение |
| **Environment** | STAGING (тестовый хаб, эмулятор термостата)                          |

#### **FR-002: Освещение с контекстным пониманием**
| **Test ID** | T-002                                                                 |
|-------------|-----------------------------------------------------------------------|
| **Requirement** | FR-002                                                               |
| **Scenario** | Автоматическое включение света при обнаружении движения в вечернее время |
| **Preconditions** | 1. Датчик присутствия установлен в коридоре<br>2. Время 19:30, естественное освещение < 10 люкс<br>3. Система в режиме "Дома" |
| **Test Steps** | **Given:** Пользователь входит в коридор<br>**When:** Датчик обнаруживает движение<br>**Then:** Свет включается на 80% яркости<br><br>**When:** Пользователь выходит из коридора<br>**Then:** Через 2 минуты свет выключается |
| **Test Data** | Яркость: 80%, таймаут: 2 мин                                          |
| **Pass Criteria** | 1. Свет включился в течение 1 сек после движения<br>2. Яркость соответствует настройке<br>3. Свет выключился через 2 мин ±10 сек |
| **Environment** | DEV (реальные устройства)                                             |

#### **FR-003: Голосовое управление на русском языке**
| **Test ID** | T-003                                                                 |
|-------------|-----------------------------------------------------------------------|
| **Requirement** | FR-003                                                               |
| **Scenario** | Распознавание команды "Включи свет в гостиной"                       |
| **Preconditions** | 1. Голосовой ассистент активирован<br>2. Пользователь авторизован<br>3. Свет в гостиной выключен |
| **Test Steps** | **Given:** Пользователь произносит "Окей, Умный дом, включи свет в гостиной"<br>**When:** Система обрабатывает команду<br>**Then:** Свет в гостиной включается<br>**And:** Ассистент отвечает "Свет в гостиной включен" |
| **Test Data** | Команда: "Включи свет в гостиной". Ожидаемый ответ: "Свет в гостиной включен" |
| **Pass Criteria** | 1. Свет включен<br>2. Ответ ассистента корректен<br>3. Время отклика < 2 сек |
| **Environment** | PROD-like (тестовый микрофон)                                         |

#### **PERF-001: Задержка управления через мобильное приложение**
| **Test ID** | T-004                                                                 |
|-------------|-----------------------------------------------------------------------|
| **Requirement** | PERF-001                                                             |
| **Scenario** | Отправка команды на изменение температуры с задержкой < 500 мс       |
| **Preconditions** | 1. 100 виртуальных пользователей одновременно подключены<br>2. Сервер под нагрузкой |
| **Test Steps** | **Given:** Нагрузочный тест JMeter с 100 параллельными запросами "установить температуру 21°C"<br>**When:** Запуск теста на 5 минут<br>**Then:** 95% запросов обрабатываются ≤500 мс |
| **Test Data** | Команда: установка температуры, 100 пользователей                    |
| **Pass Criteria** | 1. p95 latency ≤ 500 мс<br>2. Error rate = 0%<br>3. CPU сервера ≤70% |
| **Environment** | Load Zone (Yandex.Cloud)                                             |

#### **SEC-001: Локальная обработка данных**
| **Test ID** | T-005                                                                 |
|-------------|-----------------------------------------------------------------------|
| **Requirement** | SEC-001                                                              |
| **Scenario** | Попытка доступа к данным без авторизации                             |
| **Preconditions** | 1. Данные датчиков хранятся локально<br>2. Внешний IP пытается получить доступ через API |
| **Test Steps** | **Given:** Неавторизованный запрос к эндпоинту `/api/sensors`<br>**When:** Отправлен GET запрос<br>**Then:** Система возвращает HTTP 403<br>**And:** Попытка логируется в security.log |
| **Test Data** | Запрос: `GET /api/sensors`, заголовок без токена                     |
| **Pass Criteria** | 1. HTTP-код 403<br>2. Данные не возвращены<br>3. Запись в логе содержит IP и timestamp |
| **Environment** | PROD (с тестовым датчиком)                                           |
